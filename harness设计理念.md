在长运行应用程序（Long-running applications）开发中，**Harness 设计**代表了从“单次对话式编程”向“自主化、多阶段工程流”的范式转变。作为一名 Claude Code 使用与 Harness 工程专家，我将基于 **Claude Agent SDK** 的核心能力，指导你如何构建一个能够自主完成全栈软件生成、具备主观审美评估能力的 Harness 系统。

### 一、 Harness 的核心三代理架构
为了解决单代理在长程任务中容易产生的“上下文焦虑”（Context Anxiety，即模型因接近上下文极限而过早结束任务）和自我评价过度自信的问题，Harness 采用了 **Planner-Generator-Evaluator** 的三代理协作结构。

| 代理角色 | 职责定义 | 核心产出/机制 |
| :--- | :--- | :--- |
| **Planner (规划者)** | 架构设计与需求拆解。将 1-4 句的模糊需求扩展为详尽的产品规格说明书。 | 技术栈定义（如 React, FastAPI）、组件拆分、实施阶段划分。 |
| **Generator (生成者)** | 代码实施。以“冲刺（Sprint）”为单位执行开发任务，利用 Git 进行版本管理。 | 生成功能完备的组件与后端逻辑，并进行初步自我检查。 |
| **Evaluator (评估者)** | 质量裁决与主动测试。不写代码，仅负责挑刺和评分。 | 使用 Playwright MCP 进行端到端交互测试，并依据评分准则（Rubric）给出反馈。 |

### 二、 治理层建设：CLAUDE.md 与 SDK 编排
在 Harness 项目中，一致性是长程任务成功的关键。你需要通过 `CLAUDE.md` 建立工程规范，并利用 SDK 进行自动化调度。

1.  **多层级治理 (Governance)**：在项目根目录设置通用的 `CLAUDE.md` 定义全局规范，在 `/frontend` 或 `/backend` 目录下设置特定规则文件。这能实现“渐进式上下文披露”，确保代理仅获取当前任务最相关的指令，防止规则僵化。
2.  **SDK 编排与上下文重置**：利用 Claude Agent SDK 的自动压缩（Compaction）或手动上下文重置（Reset）机制。对于极长任务，手动重置可以提供一个“干净的白板”，通过结构化的“状态交接文档”将上一个会话的产出传递给下一个代理，从而彻底消除上下文焦虑。
3.  **冲刺契约 (Sprint Contract)**：在 Generator 开始编写代码前，强制其与 Evaluator 达成“什么是‘完成’”的契约。这弥补了高层级需求与可测试细节之间的鸿沟。

### 三、 攻克“AI 平庸设计”：审美量化评估
Harness 设计最显著的突破是让 AI 能够评估并追求“主观质量”。系统通过四个维度将抽象的“品味”转化为可量化的分数。

*   **设计质量 (Design Quality)**：视觉身份的连贯性。模型会被要求追求“博物馆级”的设计，而非通用的 AI 风格。
*   **原创性 (Originality)**：显式惩罚对 UI 库默认样式（如“白色卡片上的紫色渐变”）的过度依赖，鼓励自定义创意决策。
*   **工艺 (Craft)**：技术执行力检查，包括间距一致性、字体层级、色彩和谐度及对比度。
*   **功能性 (Functionality)**：可用性验证，确保用户能直观地完成核心任务。

**专家建议**：策略性地将 **Design Quality** 和 **Originality** 的权重调高。因为 AI 天然擅长技术性的“工艺”，但需要在审美风险上受到“压力推力”才能跳出平庸。

### 四、 对抗性迭代：细化与转向 (Refine vs. Pivot)
Generator 与 Evaluator 之间的关系类似于生成对抗网络（GAN）。
1.  **主动测试 (Active Testing)**：Evaluator 利用 **Playwright MCP** 像真实用户一样点击按钮、检查 API 响应和数据库状态，而非仅看静态截图。
2.  **反馈循环**：每次迭代结束后，Evaluator 给出详细批判。
3.  **战略决策**：如果评分持续上升，Generator 继续 **细化 (Refine)** ；如果评分停滞不前或低于硬性阈值，系统必须 **转向 (Pivot)** ，彻底放弃当前的审美路径，尝试全新的设计方向。

### 五、 实践中的权衡：成本与优化
长运行的 Harness 任务是极其昂贵的。例如，生成一个复杂的 2D 游戏制作器或 DAW（数字音频工作站）可能需要 4-6 小时，Token 成本在 120 美元至 200 美元之间。

为了管理运营开销，你可以采取以下优化策略：
*   **计划先行模式 (Plan-First)**：强制要求在执行复杂任务前生成 `task_plan.md`，减少无效迭代。
*   **架构简化**：随着模型能力的提升（如从 4.5 升级到 4.6），某些原本 load-bearing（承重）的 Harness 组件（如严格的冲刺拆解）可以被移除，转而利用模型的原生逻辑处理更长的上下文。
*   **子代理隔离 (Sub-agents)**：对于特定的安全审计或代码评审任务，启动独立的子代理，以保持主会话上下文的聚焦与廉价。

通过构建这一套“对抗性实验室”，你将能够驱动 AI 超越其平均水平，生产出真正具有生产质量甚至“艺术感”的全栈应用。
